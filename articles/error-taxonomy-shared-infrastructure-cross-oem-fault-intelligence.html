Error Taxonomy as Shared Infrastructure: Cross-OEM Fault Intelligence

Standardizing error semantics is the fastest path to portfolio-level anomaly detection quality.

Target audience: O&M Engineers / Reliability Teams

Every OEM has its own status vocabulary. Without a shared taxonomy, your fault analytics engine is comparing unlike events and producing noisy dispatch decisions.

Normalization Strategy

Map OEM-native states into ODSE `error_type` classes while preserving raw code in `error_code_original` for root-cause diagnostics.

Operational Impact

Teams gain portfolio-wide comparability: same alert thresholds, same cohort analysis, same false-dispatch measurement across vendor ecosystems.

Implementation Pattern

Use transform mappings for classification, then validate with semantic checks to catch inconsistent state/value combinations.

Expected Outcome

Lower triage ambiguity, faster MTTD, and cleaner prioritization queues for field teams.
