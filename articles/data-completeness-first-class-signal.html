Data Quality

Data Completeness Is a First-Class Signal (Not a Reporting Afterthought)

Missingness and timestamp integrity are not metadata trivia. They are core determinants of decision quality.

Target audience: Sustainability / Reporting / Data Teams

Teams frequently compute precision metrics on incomplete data and only discover gaps at reporting time. By then, corrective action windows are gone.

Completeness as an Operational Metric

Track expected vs observed intervals per site and per day. Report confidence bands alongside performance metrics.

Why It Changes Outcomes

Completeness errors can invert conclusions about efficiency, degradation, or compliance status when gaps are non-random.

Recommended Controls

Gate critical analytics on minimum completeness thresholds and emit explicit warnings when intervals are imputed or delayed.

Pipeline Discipline

Run schema and semantic validation before completeness scoring so you are measuring meaningful intervals, not malformed records.
